# AURA ARENA — Progressive Web App

## Overview

AURA ARENA is a fully installable PWA (Progressive Web App). It works offline, syncs in the background, and can be added to the home screen on any device.

---

## Installation Flow

### When is the install prompt shown?
After the user's **second session** (not second visit — second completed training session). This ensures they've experienced the core value before being asked to install.

### Custom Install Banner
A card that slides up from the bottom:
- App icon + name
- "Add to Home Screen for the full arena experience"
- Two buttons: "Not Now" (dismisses for 7 days) and "Install" (triggers browser install flow)

```tsx
// InstallBanner.tsx
const handleInstall = async () => {
  if (!deferredPrompt) return;
  deferredPrompt.prompt();
  const { outcome } = await deferredPrompt.userChoice;
  if (outcome === 'accepted') {
    dismissInstall();
    toast.success('AURA ARENA installed!');
  }
};
```

### Dismiss behavior
- "Not Now": `localStorage` stores timestamp, banner hidden for 7 days
- `installPromptDismissed` in Zustand store persisted via `persist` middleware

## Service Worker

Generated by `vite-plugin-pwa` using Workbox. Manual fallback at `public/sw.js`.

### Caching Strategies

| Resource | Strategy | Cache Name |
|---|---|---|
| JS, CSS, HTML, Fonts | Cache First | `static-assets` |
| Supabase API calls | Network First | `supabase-cache` (5min expiry) |
| Images (avatars, etc.) | Stale While Revalidate | `image-cache` (24h) |
| MediaPipe model files | Cache First | `mediapipe-models` (7 days) |
| Unknown | Network First | — |

### Offline Fallback
When the user navigates to a page that isn't cached and has no connection, they see `offline.html` — a styled page consistent with the app design:

```html
<!-- offline.html (in /public) -->
<div style="background:#040610; color:#f4f6ff; font-family:'DM Sans',sans-serif; ...">
  <div style="font-size:4rem">⚡</div>
  <h1>You're offline</h1>
  <p>Your last session data is saved. Connect to sync.</p>
  <button onclick="window.location.reload()">Try Again</button>
</div>
```

## Offline Capabilities

### What works offline
- Dashboard (shows cached data)
- Training session (scores computed in-browser, saved to IndexedDB)
- Reels feed (shows cached reels)
- Profile (shows persisted Zustand state)
- Arena Hub navigation

### What requires connection
- Login / sign up
- Leaderboard updates
- Posting reels
- Live battles
- Notifications from server

## Background Sync

Sessions recorded offline are stored in IndexedDB and synced when connection is restored.

### How it works

```ts
// offlineQueue.ts
const DB_NAME = 'aura-arena-offline';
const STORE = 'pending_sessions';

export const saveOfflineSession = async (session: SessionData) => {
  const db = await openDB(DB_NAME, 1, {
    upgrade(db) {
      db.createObjectStore(STORE, { keyPath: 'id' });
    },
  });
  await db.put(STORE, { ...session, pendingSync: true });
};

export const initOfflineSync = () => {
  window.addEventListener('online', async () => {
    const pending = await getPendingSessions();
    for (const session of pending) {
      await supabase.from('sessions').insert(session);
      await deletePendingSession(session.id);
    }
    if (pending.length > 0) {
      toast.success(`${pending.length} session${pending.length > 1 ? 's' : ''} synced`);
    }
  });
};
```

`initOfflineSync()` is called once in `main.tsx` after the app mounts.

## Push Notifications

Infrastructure is ready; server implementation is pending for production.

### Client-side setup
```ts
// main.tsx
if ('serviceWorker' in navigator && 'PushManager' in window) {
  navigator.serviceWorker.ready.then(registration => {
    // Store registration for later use when user opts in
    window.__swRegistration = registration;
  });
}
```

### Permission Request Flow
1. User enables a notification type in Settings
2. Custom in-app explanation modal appears (what will be notified, why)
3. User accepts → `Notification.requestPermission()` called
4. If granted → subscribe to push server (FCM/VAPID)
5. Store subscription in Supabase `push_subscriptions` table

### Notification Types (ready to send)
- Achievement unlocked
- Tier advancement  
- Streak reminder (last 2 hours of day)
- Weekly performance summary
- New season start
- Reel likes milestone

## Apple/iOS Notes

- iOS Safari: `display: standalone` works, but no `beforeinstallprompt` event
- iOS install: user must manually use "Add to Home Screen" from Share menu
- The install banner shows on iOS with instructions to use the Share button
- `apple-touch-icon` set to `/icons/icon-192.png` in `index.html`
- `apple-mobile-web-app-status-bar-style: black-translucent` for edge-to-edge

## Android Notes

- Chrome/Edge on Android: full PWA support including install prompt, push notifications
- Samsung Internet: supported
- The install prompt fires automatically after user engagement criteria met
