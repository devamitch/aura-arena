# AURA ARENA — Complete Setup Guide

## Prerequisites

| Tool | Version |
|---|---|
| Node.js | ≥ 20.x (LTS) |
| npm | ≥ 10.x |
| Git | Any recent |
| A modern browser | Chrome 120+ / Safari 17+ / Firefox 121+ |

---

## 1. Clone & Install

```bash
git clone https://github.com/your-org/aura-arena.git
cd aura-arena
npm install
```

This installs ~95 dependencies including React 19, Framer Motion 12, Zustand 5, Supabase JS, MediaPipe, TanStack Query, and all tooling.

---

## 2. Supabase Setup

### 2a. Create a Supabase project
1. Go to [supabase.com](https://supabase.com) → New Project
2. Choose a name (e.g., `aura-arena`), set a database password, pick a region close to your users
3. Wait ~2 minutes for provisioning

### 2b. Run the database schema
1. Go to **SQL Editor** in your Supabase dashboard
2. Click **New Query**
3. Copy the entire contents of `supabase/schema.sql`
4. Click **Run**

This creates all tables, views, RLS policies, indexes, and triggers.

### 2c. Configure Google OAuth
1. In Supabase: **Authentication** → **Providers** → **Google** → Enable
2. Go to [Google Cloud Console](https://console.cloud.google.com) → APIs & Services → Credentials
3. Create OAuth 2.0 Client ID (Web application)
4. Authorized JS origins: `http://localhost:5173`, `https://your-domain.com`
5. Authorized redirect URIs: `https://your-project-id.supabase.co/auth/v1/callback`
6. Copy Client ID and Client Secret
7. Paste both into Supabase Google provider settings
8. Save your Google Client ID for the `.env.local` file

### 2d. Enable Realtime
1. Supabase Dashboard → **Database** → **Replication** (or **Realtime**)
2. Enable realtime for: `notifications`, `reels`, `reel_likes`, `battles`

## 3. Environment Variables

```bash
cp .env.example .env.local
```

Edit `.env.local`:

```env
# Supabase (from Project Settings → API)
VITE_SUPABASE_URL=https://your-project-id.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

# Google OAuth (from Google Cloud Console)
VITE_GOOGLE_CLIENT_ID=123456789-abc.apps.googleusercontent.com

# Gemini AI (from https://aistudio.google.com/apikey)
VITE_GEMINI_API_KEY=AIzaSyB...
```

> **Security note**: The Gemini API key is used client-side. For production, consider proxying through a Supabase Edge Function to protect your key from abuse.

## 4. Development Server

```bash
npm run dev
```

Opens at `http://localhost:5173`

### Hot reload works for:
- All React components
- CSS / Tailwind classes
- Store logic
- Hook changes

### Does NOT hot-reload (requires manual refresh):
- `vite.config.ts` changes
- `tailwind.config.js` changes
- Environment variable changes

## 6. Deployment

### Vercel (Recommended)
```bash
npx vercel --prod
```

Or connect your GitHub repo to [vercel.com](https://vercel.com) for automatic deployments.

Add in Vercel dashboard → Project → Settings → Environment Variables:
- All 4 env vars from your `.env.local`

Create `vercel.json` (already included):
```json
{
  "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }]
}
```

### Netlify
```bash
npx netlify deploy --prod --dir dist
```

### Cloudflare Pages
```bash
npx wrangler pages deploy dist
```

The `public/_redirects` handles SPA routing automatically on Netlify and Cloudflare.

## 7. PWA Setup

For full PWA with install prompt, the app must be served over HTTPS. Vercel/Netlify provide this automatically.

### Generating App Icons
The app references icons at `/icons/icon-*.png`. Generate them from your logo:
```bash
# Using PWA Asset Generator (one-time)
npx pwa-asset-generator logo.png ./public/icons --manifest public/manifest.json
```

Or use [PWA Builder](https://www.pwabuilder.com/imageGenerator) to generate all sizes.

## 8. Optional: MediaPipe Model Files

For production, MediaPipe models can be self-hosted for reliability:

```bash
# Create models directory
mkdir -p public/models/mediapipe

# Download pose landmarker
curl -o public/models/mediapipe/pose_landmarker_full.task \
  https://storage.googleapis.com/mediapipe-models/pose_landmarker/pose_landmarker_full/float16/latest/pose_landmarker_full.task

# Download hand landmarker
curl -o public/models/mediapipe/hand_landmarker.task \
  https://storage.googleapis.com/mediapipe-models/hand_landmarker/hand_landmarker/float16/latest/hand_landmarker.task

# Download face landmarker
curl -o public/models/mediapipe/face_landmarker.task \
  https://storage.googleapis.com/mediapipe-models/face_landmarker/face_landmarker/float16/latest/face_landmarker.task
```

Update `src/lib/mediapipe/allTasks.ts` with the local model paths when deploying.

## 11. Common Issues

### Camera not working in dev
- Chrome requires HTTPS or `localhost` for `getUserMedia`. The dev server at `localhost:5173` is fine.
- In Safari, camera permissions must be granted per-origin.
- Firefox may require `media.navigator.enabled: true` in `about:config`.

### Gemini API quota exceeded
- The app uses `gemini-2.0-flash` by default (cheapest). Upgrade to `gemini-2.5-flash` for better quality.
- The 5-minute TTL cache prevents most redundant calls.
- Set `VITE_GEMINI_API_KEY` to a key with billing enabled for production.

### Supabase realtime not updating
1. Check that realtime is enabled for the table in Supabase dashboard
2. Check that `ALTER PUBLICATION supabase_realtime ADD TABLE ...` was run
3. Check RLS policies — realtime respects them

### Offline sync not working
- Requires the app to be served from a service worker context (production build or `npm run preview`)
- `navigator.onLine` must be accurate — some browser/network configurations can mask offline status

### Tailwind classes not applying
- Check `tailwind.config.js` content array includes your file path
- Run `npm run dev` to restart the Vite dev server after config changes
